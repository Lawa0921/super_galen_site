---

## 1. 目標與動機

在 Dungeon Arcade（`/games`）新增第二款單機小遊戲：一款 **roguelite 風格的炸彈超人**。參考《炸彈超人》《淡水阿給》那類俯視 2D 格子動作遊戲。

核心訴求：

- **單機優先**：純前端、無後端，契合靜態 Vercel 部署（不做線上連線）。
- **延續 tetris 的工程做法**：核心引擎為純 TypeScript、零 Pixi 依賴、全單元測試（TDD）；渲染層用 PixiJS。
- **重視美術實作**：用 Gemini（nanobanana）預先生成像素美術素材，程式負責合成、動畫、特效，而非用 DOM `<div>` 拼盤面。
- **無限重玩**：隨機生成樓層，越深越難，拚「炸到第幾層 / 分數」，美術量有界。

## 2. 視覺方向（已定案：B 經典 16-bit 像素）

三種方向已生成 mockup 比對（存於 `docs/superpowers/specs/dungeon-bomber-mockups/`）：

| 檔案 | 方向 | 結果 |
|---|---|---|
| `style-A-neon-painterly.webp` | 全 painterly 霓虹（最貼現有 tetris） | 未採用：牆/可炸箱辨識度偏低 |
| `style-B-pixel-retro-CHOSEN.webp` | **經典 16-bit 像素（採用）** | 辨識度最高、最懷舊；木箱(可炸) vs 灰柱(不可炸)一眼分明 |
| `style-C-hybrid.webp` | painterly 地牢底 + 霓虹遊戲件 | 未採用：折衷但較費工 |

**採用 B 的理由**：俯視動作遊戲最怕「看不清牆/可炸箱/危險爆風」，像素風辨識度最佳，也最貼炸彈超人本體。

**取捨**：B 的扁平灰地板與現有 painterly 賽博地牢調性有落差。緩解做法：深層樓層用「換色調主題」（青/洋紅/暖橘 tint）讓它仍屬於 Dungeon Arcade 家族；大廳卡片與 HUD 沿用既有霓虹框與 Press Start 2P 字體維持一致。

**主角差異化**：主角造型客製化（做出和原版不同的辨識度）排在素材定稿後再做，不阻擋 v1。

## 3. 玩法設計（已通過）

### 3.1 核心循環

進入隨機生成的格子樓層 → 放炸彈炸軟箱（可能掉道具）→ **炸掉該層所有敵人** → 出現往下樓梯/傳送門 → 踩下去進入更深一層 → 重複直到死亡 → 結算分數 → 可重來。

### 3.2 規則

- **過關條件**：清空該層所有敵人後，出口出現，踩入即下一層。
- **生存**：**一擊死**。被自己或敵人的爆風炸到、或被敵人碰到即死。
- **命數**：每趟 **3 條命**。死亡 −1 命，原地重生並短暫無敵閃爍；命數歸零 → 該趟結束。
- **護盾道具**：撿到後可擋一次傷害（吸收後消失）或提供短暫無敵（實作時二擇一，spec 預設「擋一次」）。
- **計分**：分數 = f(到達層數、擊殺敵人數、撿取道具、剩餘時間獎勵)。具體公式於實作期定，集中在 `scoring.ts`。
- **死亡結算**：顯示到達層數與分數，提供「重新開始」。

### 3.3 移動模型（技術抉擇，已選）

**格子鎖定 + 視覺補間**：邏輯上一格一格走（如 NES 原版），畫面用 tween 平滑滑過去。引擎乾淨、好測、retro 手感扎實。（未採用「自由像素移動」，因碰撞與測試複雜度高。）

## 4. 範圍（Scope）

### 包含（In scope, v1）
- `/games/bomber`：全螢幕遊戲頁，掛載 PixiJS app（桌機鍵盤操作）。
- `/games` 大廳：把一張 `COMING SOON` 卡換成本遊戲入口。
- 隨機樓層生成（seeded RNG，可重現、可測）。
- 炸彈機制：放置、計時引爆、十字爆風、**連鎖引爆**（爆風波及其他炸彈即時引爆）。
- 一擊死 + 3 命 + 護盾容錯。
- **道具 4 種**：火力＋（爆風範圍）、炸彈＋（同時可放數量）、加速、護盾。
- **敵人 2 種**：徘徊型（隨機走）、追擊型（朝玩家走，簡易格子尋路）。深層調整數量/速度。
- HUD：命數、層數、目前道具狀態、分數。
- 死亡結算與重新開始。
- 像素美術素材（生成 → 去背 → 切片 spritesheet → nearest-neighbor 渲染）。
- Web Audio 合成音效（放彈、爆炸、撿道具、死亡、下樓）。
- i18n：標題 / HUD / 結算字串（5 語），沿用站上慣例。
- 桌機優先；MVP 為預設語言頁。

### 不包含（YAGNI，延後）
- 觸控操作（手機虛擬方向鍵 + 炸彈鈕）→ 之後補。
- 線上 / 本機雙人對戰（本作純單機）。
- 進階道具：踢炸彈、遙控引爆、穿牆爆風。
- 第三種以上敵人、Boss 關。
- 永久解鎖 / meta 升級系統。
- BGM（先只做合成 SFX）。
- `[lang]/games` 多語系路由（站上慣例後補）。

## 5. 架構：邏輯與渲染分離（沿用 tetris）

關鍵原則：**核心引擎純 TypeScript、零 Pixi 依賴、全單元測試（TDD）**；渲染層 PixiJS 靠 dev server 人工驗收。

```
src/scripts/games/bomber/
  engine/    純 TS（可測）
  render/    PixiJS 繪圖
  input/     鍵盤輸入
  audio/     Web Audio 合成音效
  assets.ts  像素貼圖載入（nearest-neighbor）
src/pages/games/bomber.astro   遊戲頁
src/pages/games/index.astro    大廳入口（改一張卡）
```

### 5.1 核心引擎（`engine/`，純 TS，可測）

| 模組 | 職責 |
|---|---|
| `constants.ts` | 格子尺寸、計時、道具/敵人參數、計分表（純常數） |
| `types.ts` | Tile/Bomb/Blast/Enemy/PowerUp/PlayerState/FloorState 等型別 |
| `rng.ts` | seeded RNG（可重現生成與測試）；可沿用/參考 tetris `rng.ts` |
| `board.ts` | 格子盤面：tile 種類（地板/不可炸柱/可炸軟箱）、查詢、破壞軟箱 |
| `generate.ts` | 樓層生成：外牆 + 固定柱子格狀陣列 + 隨機軟箱（出生點留空）+ 敵人/道具佈點（依層數調參） |
| `bomb.ts` | 炸彈放置、計時、引爆；**連鎖引爆**解析 |
| `blast.ts` | 十字爆風範圍解析（遇不可炸柱阻擋、炸毀軟箱、波及敵人/玩家/其他炸彈） |
| `player.ts` | 玩家格子移動（含補間目標）、放彈、撿道具、受傷/死亡/無敵 |
| `enemy.ts` | 敵人 AI：徘徊型、追擊型（格子尋路）、移動節奏 |
| `powerup.ts` | 道具效果套用（火力/炸彈/加速/護盾）與狀態 |
| `game.ts` | 單機狀態機：floor 進行、命數、過關→下樓、結算；對外 `tick(dt)` 與 `InputAction` |
| `scoring.ts` | 分數計算 |

### 5.2 渲染層（`render/`，PixiJS）
- `PixiStage`：app 建立、resize、主迴圈接 `engine.tick`。
- `TilemapView`：地板/柱/軟箱 tile 繪製（含破裂動畫）。
- `Sprites`：玩家（4 向走路）、敵人、炸彈（脈動）、爆風（中心/臂/端點）、道具、樓梯。
- `HudView`：命數、層數、道具、分數。
- `Effects`：爆炸粒子、震屏、撿取/死亡特效。

### 5.3 輸入（`input/`）
- `keymap.ts`：方向鍵/WASD → 移動；空白鍵（或 Z/J）→ 放彈。沿用 tetris 的 `InputController` 模式（`KeyboardEvent.code`）。

### 5.4 音效（`audio/`）
- `SoundManager`：Web Audio 合成 SFX（放彈、引信、爆炸、撿道具、受傷、下樓、結算）。沿用 tetris 做法。

## 6. 美術實作計畫（B 像素風）

### 6.1 素材清單
| 類別 | 內容 |
|---|---|
| 地形 | 地板磚、不可炸柱、可炸軟箱（含 2–3 幀破裂） |
| 主角 | 4 向走路各 2–4 幀（idle + walk）；造型差異化定稿後做 |
| 敵人 | 徘徊型、追擊型，各走路數幀 |
| 炸彈 | 脈動 2–3 幀 + 引信火花 |
| 爆風 | 中心、臂、端點，各數幀（出現→消散） |
| 道具 | 火力＋、炸彈＋、加速、護盾 4 個圖示 |
| 場景 | 下樓梯/傳送門（出口）、HUD 框元素 |

### 6.2 產製管線
1. 用 nanobanana 生**單一素材**（像素風、純色/透明或黑底）。
2. 亮度轉真 alpha 去背（沿用既有素材去背手法，避免黑方框）。
3. 切片成 spritesheet（或個別 webp）。
4. 輸出 webp；Pixi 以 `scaleMode = 'nearest'` 渲染保留像素硬邊。
5. 深層樓層以色調 tint 變化主題，重用同一套素材。

放置路徑：`public/assets/games/bomber/`。

## 7. 測試策略

- **引擎（Vitest，TDD）**：每個 engine 模組先寫測試。重點覆蓋：
  - 樓層生成（seed 可重現、出生點淨空、柱子陣列正確、軟箱/敵人不重疊）。
  - 爆風解析（範圍、被柱阻擋、炸毀軟箱、波及玩家/敵人）。
  - **連鎖引爆**（爆風觸發鄰近炸彈即時連爆）。
  - 道具效果（火力/炸彈上限/加速/護盾擋傷）。
  - 一擊死、命數遞減、過關→下樓、計分。
  - 敵人 AI（徘徊不卡牆、追擊朝玩家）。
- **E2E（Playwright）**：solo smoke——遊戲開得起來、能移動、能放彈炸掉軟箱、清關後能下樓。

## 8. v1 MVP 切割（建議實作階段）

1. **Phase 1 — 引擎骨架**：board + generate + types + rng + 基本 `game.tick`（移動）。（純測試）
2. **Phase 2 — 炸彈與爆風**：bomb + blast + 連鎖引爆 + 軟箱破壞。（純測試）
3. **Phase 3 — 渲染最小可玩**：PixiStage + Tilemap + 玩家 sprite + 炸彈/爆風，鍵盤可操作（用佔位素材）。
4. **Phase 4 — 敵人與一擊死/命數**：enemy AI + 受傷死亡 + 過關下樓 + HUD。
5. **Phase 5 — 道具與計分**：4 道具 + 護盾 + scoring + 死亡結算。
6. **Phase 6 — 美術定稿**：替換正式像素素材、動畫、特效、音效、深層 tint。
7. **Phase 7 — 收尾**：大廳入口、i18n、Playwright smoke、效能驗收。
