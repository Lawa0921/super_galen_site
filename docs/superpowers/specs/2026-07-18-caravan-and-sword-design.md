# 《商隊與劍》(Caravan & Sword) 設計規格

日期：2026-07-18｜狀態：使用者已核可設計方向，本文件為正式規格
路徑：`/games/caravan`｜遊戲廳卡片標題：`CARAVAN & SWORD`（名稱為暫定可再議）

## 0. 一句話定位

單機文字跑團 RPG：玩家繼承沒落商隊，在奇幻大陸上經營商隊與傭兵團——接委託、跑貿易路線、深入迷宮，以 d20 擲骰決定敘事走向，永續存檔累積等級、財富、裝備與聲望。

## 1. 已確認的設計決策（使用者逐項選定）

| 決策點 | 選定 |
|--------|------|
| 呈現形式 | 文字跑團風（敘事段落＋選項＋擲骰檢定），非地圖/戰棋/roguelike |
| 進度結構 | 永續角色＋遠征制：城鎮為永續基地，每趟遠征 15–30 分鐘 |
| 戰鬥 | 隊伍骰子回合制（敘事畫面內，非獨立戰鬥場景） |
| 語言 | 敘事全繁體中文；介面保留少量英文 RPG 術語（HP/DC/Lv）；不進 i18n 五語系統 |
| 美術 | **日本動漫平塗風插圖**（cel-shading、乾淨線稿、平塗上色）；介面為乾淨文庫本排版＋淡色紙質（原油畫古書案已被使用者修訂） |
| 範圍策略 | Phase 1 完整核心循環：五大系統（事件/戰鬥/迷宮/商隊/傭兵）皆含基礎版，後續以內容資料擴充 |

## 2. 核心循環

```
城鎮（整備）
 ├─ 酒館：招募傭兵（池每次歸返輪替）
 ├─ 市集：買賣商品/裝備（價格隨城鎮與事件浮動）
 ├─ 委託板：選遠征（貿易路線／迷宮探索／傭兵委託）
 └─ 倉庫/療養所：存放與復原
      ↓ 出發
遠征（事件鏈）
 ├─ 事件卡 ×N：敘事段落 → 2–3 選項 → d20 檢定 → 成敗結果
 ├─ 戰鬥遭遇：隊伍骰子回合制
 ├─ 迷宮節點（若目標為迷宮）：逐層房卡選擇
 └─ 隨時可撤退（有代價）
      ↓ 歸返
結算：戰利品入庫、經驗升級、薪餉支付、聲望變動、招募池輪替、自動存檔
```

## 3. 系統規格

### 3.1 屬性與檢定

- 五屬性：力量／敏捷／智力／魅力／體質。
- 檢定：`d20 + 屬性調整值 + 情境加成 vs DC`。關鍵擲骰有骰子動畫與音效。
- 隊伍優勢：特定職業在隊提供對應檢定加成（例：游俠→野外類 +2）。

### 3.2 事件系統（資料驅動）

事件卡 schema（TS 型別，內容檔在 `data/events.ts`）：

```ts
interface EventCard {
  id: string;
  context: { terrain?: Terrain[]; routeIds?: string[]; dungeonIds?: string[]; minReputation?: number };
  weight: number;                 // 加權抽選
  conditions?: Flag[];            // 旗標前置
  title: string; body: string;    // 繁中敘事
  art?: string;                   // 插圖資產路徑
  options: Array<{
    label: string;
    requirement?: { job?: Job; item?: ItemId };   // 顯示條件
    check?: { stat: Stat; dc: number };           // 無 check 則直接走 outcome
    success: Effect[]; failure?: Effect[];
  }>;
}
type Effect = 金錢增減 | 物品得失 | HP/傷勢 | 旗標設置 | 聲望變動 | 觸發戰鬥 | 商品價格擾動;
```

- 同一路線重跑抽到不同組合；旗標讓少量事件可以串成短鏈。
- Phase 1 目標 40 張事件卡。

### 3.3 戰鬥（隊伍骰子回合制）

- 先攻：`d20 + 敏捷調整`，排成行動序。
- 我方回合：每名隊員 3 個招式（職業 2＋武器 1），選招式與目標 → 命中 `d20+修正 vs 防禦` → 傷害骰（武器決定，如 2d6+力量）→ 一行敘事描述。
- 敵方意圖：下回合行動前一回合預告（「呪術師詠唱中——下回合範圍攻擊」）。
- 撤退：任何我方回合可選，代價＝掉落部分戰利品＋殿後者受一次攻擊。
- 歸零處理：傭兵→體質檢定，成功＝重傷（缺席接下來 1–2 趟遠征療養；Phase 1 無日曆系統，時間一律以遠征趟數計），失敗＝**永久死亡**；主角必定重傷不死（遊戲不結束）。

### 3.4 迷宮

- 逐層房卡：每層抽 2–3 張（戰鬥／寶箱／事件／休息／未知），選一張進入。
- 深度係數：層數越深，敵人強度與獎勵倍率越高；隨時可撤（保留已得戰利品、放棄「完探加成」）。
- 底層：頭目戰＋遺寶（獨特裝備）。Phase 1 兩座迷宮，各 5–7 層。

### 3.5 商隊（Phase 1 簡版）

- 各城鎮商品基準價 ±浮動（事件效果可擾動：例「礦坑封閉→鐵價上漲」）。
- 載貨量由馬車等級決定；貿易路線本身是一場遠征——押貨越多，遇襲事件損失越痛（風險/報酬掛鉤）。
- 介面：城鎮市集顯示「此地買價/他城參考價」。

### 3.6 傭兵團

- 酒館招募池 3–5 人，每次歸返輪替；出征隊上限 4（含主角）。
- 每趟遠征付薪餉（等級越高越貴）；受傷/死亡永久（見 3.3）。
- 職業四種（Phase 1）：劍士（前排/力量）、游俠（檢定輔助/敏捷）、法師（範圍/智力）、教士（治療/魅力）。

### 3.7 成長與裝備（M5 已完成）

- 升級：經驗表 → 屬性點＋新招式（每職業一條 5 級招式線）。✅
- 裝備三欄：武器／護甲／飾品；來源＝市集、迷宮遺寶、事件獎勵。✅（M5：equipItem/unequipItem/equipmentBonus、武器招取代 moves[0]、存檔 v5）
- 商隊聲望：完成委託累積 → 解鎖更高階委託與新城鎮。✅（M5：reputation ≥40 霧嶺古道／≥60 鹽晶洞窟、第四鎮鹽泉城）

## 4. 技術架構

### 4.1 前端

- **UI 為 DOM/HTML＋CSS**（不用 Pixi/Canvas）：文字遊戲以排版為主體，動漫平塗插圖為 `<img>` 資產卡；頁面 `src/pages/games/caravan.astro`（樣式照既有遊戲頁模式放頁內）。
- 遊戲廳 `src/pages/games/index.astro` 註冊卡片 `CARAVAN & SWORD`。
- 音效：Web Audio 合成（擲骰、翻頁、金幣、劍擊）；BGM 後續以 `gemini-music-gen` 產出、`ArcadeBgm` 模式接入（preload=none＋手勢後播放）。

### 4.2 引擎模組（`src/scripts/games/caravan/`）

| 模組 | 職責 |
|------|------|
| `rng.ts` | 種子化隨機（mulberry32 或同級），所有骰與抽選經此——測試可重現 |
| `check.ts` | 檢定結算（骰值、修正、優勢、成敗判定） |
| `combat.ts` | 戰鬥狀態機（先攻、回合、招式結算、撤退、歸零處理） |
| `expedition.ts` | 遠征狀態機（事件抽選、選項結算、迷宮層推進、結算） |
| `economy.ts` | 價格表、浮動、買賣、薪餉 |
| `roster.ts` | 傭兵池、招募、傷勢/死亡、升級 |
| `save.ts` | localStorage 存檔，版本化 schema＋遷移函式；每事件結算與歸返自動存 |
| `data/*.ts` | events / items / enemies / jobs / towns / goods / dungeons —— 純資料，擴充內容不改引擎 |

### 4.3 存檔

- localStorage key `caravan-save-v1`；schema 帶 `version` 欄位，`save.ts` 內建逐版遷移（Phase 2 加系統時不清檔）。
- 匯出/匯入字串（base64）作備援（Phase 1 就做，很便宜）。

## 5. 美術規格（日本動漫平塗）

- 風格關鍵字：cel-shading 平塗、乾淨線稿、明快配色、輕小說插畫感；場景卡橫式 16:10、敵人立繪直式。
- Phase 1 資產清單：場景/事件插圖 ~20 張、敵人立繪 12 張、職業頭像 4＋主角 1、城鎮外觀 3、UI 裝飾（骰子、金幣、紋章）若干。
- 產圖：`nanobanana-image-gen` skill，先產風格錨定圖 1–2 張經使用者確認後，以 `--ref` 鏈維持整組一致性（既有 memory 慣例）。
- 介面：乾淨文庫本排版、淡色紙質底、細線分隔——**不用**油畫古書紋理（與平塗畫風衝突）。

## 6. 測試策略（全域 TDD 鐵則適用）

- 引擎全部種子化：`check`/`combat`/`economy`/`expedition`/`save` 各模組 vitest 先紅後綠；重點案例——檢定邊界（nat 1/20）、戰鬥歸零分支（重傷 vs 死亡）、經濟不變量（買賣不產生無中生有）、存檔遷移（v1→v2 假想遷移測試）。
- e2e（chromium）：`tests/e2e/caravan.spec.ts`——開新檔 → 城鎮招募 → 出遠征 → 完成一個事件檢定 → 一場戰鬥 → 歸返結算 → 重新整理後存檔仍在。
- 完成判準照 `docs/claude/verify.md`（vitest 全套＋相關 e2e＋4321/自有埠驗收 URL）。

## 7. Phase 1 內容量（首發範圍）

1 地區：3 城鎮、4 條貿易路線、2 座迷宮（5–7 層）；40 事件卡；12 敵人（含 2 頭目）；30 物品/商品；4 職業 ×5 級招式線；委託 3 型（貿易護送/迷宮探索/討伐）。

## 8. 後續 Roadmap（非本次範圍）

- Phase 2：新地區與城鎮解鎖、商隊雇員（車夫/斥候）與路線專屬事件、傭兵聲望階層、事件鏈任務（多段旗標劇情）。
- Phase 3（遠期）：獨立戰棋戰鬥場景、成就與主站 RPG 介面聯動。

## 9. 風險與開放問題

- **名稱**：《商隊與劍》/`CARAVAN & SWORD`/slug `caravan` 為暫定，實作前可再議。
- **內容產能**：40 張事件卡的繁中敘事是最大工作量；以 schema 先行、批次撰寫、事件可組合設計來攤平。
- **平塗風格一致性**：先錨定 1–2 張風格圖經使用者核可再批量產出。
- **平衡**：經濟與戰鬥數值首發後必然要調——數值全在 data 檔，調整不動引擎。
